클라우드 기본 면접

클라우드 기본 면접은 서비스 이름 암기가 아니라 네트워크 경계, 권한, 운영 책임, 비용을 설명하는 능력을 봅니다.

CLD-01

VPC·Subnet·Routing

VPC는 클라우드 안의 논리적 네트워크 경계입니다. public subnet은 인터넷에서 접근 가능한 리소스, private subnet은 외부에서 직접 접근하지 않는 app server나 DB를 두는 영역으로 설명할 수 있습니다.

질문답변 방향
public/private subnet 차이는?route table이 Internet Gateway로 나가는 경로를 갖는지와 외부 접근 정책으로 설명합니다.
DB는 어디에 두나요?private subnet에 두고 app 또는 bastion/VPN 같은 제한된 경로에서만 접근하게 합니다.
CLD-02

NAT Gateway·LB

NAT Gateway는 private subnet의 리소스가 외부 API나 패키지 저장소로 outbound 요청을 보낼 때 사용합니다. 외부에서 private 리소스로 직접 들어오는 경로는 열지 않습니다.

구성요소역할
Internet Gatewaypublic subnet과 인터넷 사이의 입출구입니다.
NAT Gatewayprivate subnet의 outbound 인터넷 접근을 제공합니다.
Load Balancer트래픽 분산, health check, TLS termination, rolling deploy 안정성에 기여합니다.
CLD-03

Security Group

Security Group은 인스턴스 또는 리소스 단위의 stateful firewall입니다. “DB 5432를 0.0.0.0/0에 연다”가 아니라 app security group에서만 접근하도록 source를 제한하는 식으로 말해야 합니다.

질문답변 방향
Security Group과 NACL 차이는?SG는 stateful 리소스 단위, NACL은 stateless subnet 단위 규칙으로 설명합니다.
운영 접근은?VPN, SSM, bastion, MFA, audit log를 기준으로 제한합니다.
CLD-04

IAM·Secret

IAM은 사용자와 서비스가 어떤 리소스에 어떤 action을 할 수 있는지 정의합니다. 최소 권한 원칙을 적용하고, 장기 access key보다 role 기반 임시 credential을 우선합니다.

면접 답변CI/CD, app server, batch worker마다 역할을 분리하고 필요한 action만 허용합니다. secret은 코드와 이미지에 넣지 않고 secret manager에 저장하며 rotation, audit, 접근 권한을 관리합니다.
CLD-05

Managed 서비스

managed DB, object storage, CDN, queue, serverless는 직접 운영 부담을 줄이지만 비용, vendor lock-in, 세부 제어 제한이 생깁니다. 면접에서는 “왜 managed를 쓰는가”와 “어떤 책임은 여전히 우리에게 남는가”를 함께 말합니다.

서비스선택 기준
managed DBbackup, patch, replication, failover를 위임하되 schema, index, query, 접근 제어는 직접 관리합니다.
object storage정적 파일, 업로드 파일, backup 저장에 적합하며 presigned URL과 권한을 봅니다.
CDN정적 자산과 public read 콘텐츠의 latency와 origin 부하를 줄입니다.
CLD-06

비용·관측성

cost estimation은 인스턴스 비용만 계산하는 것이 아니라 compute, DB, storage, network egress, NAT Gateway, 로그 저장, managed service 요청 수를 함께 보는 작업입니다.

질문답변 방향
비용이 갑자기 늘면?egress, NAT, 로그 폭증, auto scaling, DB I/O, 잘못된 retry를 확인합니다.
관측성은?metric, log, trace, alert, dashboard를 서비스 SLO와 연결합니다.
