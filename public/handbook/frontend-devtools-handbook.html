DevTools 사용법
핸드북

DevTools는 문제를 추측에서 측정으로 바꾸는 도구입니다. 좋은 프론트엔드 디버깅은 증상을 재현하고, 레이어를 나누고, 가장 싼 가설부터 제거하는 과정입니다.

TOOLS : Chrome DevTools · React DevTools
DT-01

디버깅 루틴

DevTools를 열기 전에 문제를 한 문장으로 정의합니다. "느리다"가 아니라 "검색어 입력 후 목록 갱신까지 600ms 걸린다"처럼 관찰 가능한 증상이어야 합니다.

DEBUG LOOP재현 조건 고정
증상과 기대 동작 기록
Network로 데이터 문제 배제
Console로 runtime error 확인
Elements로 DOM/CSS 확인
Performance/Profiler로 비용 측정
수정 후 같은 조건에서 비교
HYPOTHESIS

한 번에 여러 것을 고치지 마세요. 하나의 가설을 세우고, DevTools로 근거를 확인하고, 수정 후 같은 조건에서 비교합니다.

DT-02

Elements·Styles

Elements 패널은 DOM과 CSS cascade를 확인하는 곳입니다. UI 깨짐은 대부분 여기서 원인을 찾습니다.

기능사용 상황확인할 것
Computed왜 이 스타일이 적용됐는가최종 계산값, 상속, overridden rule
Box Model간격/크기 문제content, padding, border, margin
Force statehover/focus 상태 확인focus-visible 스타일 누락
Accessibility pane보조기술 이름 확인accessible name, role, ignored 여부
DT-03

Network

Network는 API와 리소스 로딩 문제를 분리하는 첫 관문입니다.

볼 것의미판단
StatusHTTP 결과401/403/404/500을 구분
Waterfall요청 시간 분해DNS, TTFB, download 중 어디가 느린지 확인
Initiator요청을 만든 코드중복 요청의 출처 추적
Disable cache캐시 없는 조건초기 방문 성능 재현
Throttling느린 네트워크 시뮬레이션실사용 환경 검증
CACHE CONFUSION

개발 중 cache 때문에 문제가 사라지거나 생길 수 있습니다. 재현 조건을 명확히 하려면 Disable cache, hard reload, incognito를 구분해서 사용합니다.

DT-04

Performance·Memory

Performance 패널은 main thread의 실제 비용을 보여줍니다. Memory 패널은 누수가 의심될 때 사용합니다.

패널확인대표 문제
Performancelong task, scripting, rendering, painting입력 지연, scroll jank
Performance monitorCPU, JS heap, DOM nodesDOM 증가, heap 증가 추세
Memory snapshot객체 유지 경로unmounted 컴포넌트 참조 유지
Allocation timeline상호작용 중 메모리 할당반복 작업에서 누적되는 객체
성능 진단 루틴기준
CPU throttling고사양 개발 머신 기준을 피하고 중저가 기기 조건으로 재현
Long task 분해scripting, rendering, painting 중 어디가 긴지 flame chart에서 확인
Lighthouse vs PerformanceLighthouse는 lab 점수와 힌트, Performance trace는 특정 상호작용 원인 분석에 사용
React Profiler 결합Chrome trace에서 긴 작업을 찾고 React Profiler에서 어떤 컴포넌트 render가 원인인지 연결
Memory 재현동일 동작 반복 후 heap/DOM node 증가 추세 확인
DT-05

Application·Security

Application 패널은 storage, cookie, cache, service worker를 보는 곳입니다. Security 패널은 HTTPS와 인증서 문제를 확인합니다.

영역확인할 것
CookiesHttpOnly, Secure, SameSite, domain/path
Local Storage민감 정보 저장 여부
Service Worker등록 상태, update, cache 영향
Cache Storage오래된 asset이 남아 있는지
Securitymixed content, certificate, TLS 상태
보안/캐시 재현 루틴확인
CSP violationConsole과 Network에서 blocked URI와 directive 확인
Mixed contentSecurity 패널에서 HTTPS 페이지의 HTTP 리소스 차단 확인
Cookie 정책SameSite, HttpOnly, Secure, domain/path가 기대와 맞는지 확인
Service Workerupdate/unregister 후 재현, cache storage 삭제 후 stale cache 여부 확인
StoragelocalStorage/sessionStorage에 token이나 민감 정보가 남는지 확인
DT-06

React DevTools는 컴포넌트 트리, props, state, render 비용을 확인하는 도구입니다.
