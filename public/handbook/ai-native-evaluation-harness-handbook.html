EVAL-LAB-01

하네스 구조

EVALUATION HARNESS dataset loader → baseline runner → candidate runner → metric calculator
→ judge calibration → threshold gate → failure clustering → report
대상평가 질문
prompt출력 계약과 정책을 지키는가
retrieval정답 근거가 top-k 안에 들어오는가
tool도구 호출 인자와 승인 정책이 맞는가
policy거절, 마스킹, 권한 경계를 지키는가

EVAL-LAB-04

Judge calibration

LLM-as-Judge는 빠르지만 최종 권위가 아닙니다. 사람 라벨 holdout과의 agreement를 추적해야 합니다.

위험대응
긴 답변을 더 좋게 평가claim-level rubric 사용
citation 오류를 놓침evidence span 비교 추가
judge 모델 변경으로 점수 분포 변화judge도 versioned dependency로 관리
