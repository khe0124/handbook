AX 인터랙션과
마이크로인터랙션

AX 디자인에서 인터랙션은 “AI가 뭔가 하는 느낌”을 꾸미는 장식이 아닙니다. 사용자가 시스템의 상태를 이해하고, AI 결과를 검토하고, 위험한 자동화를 멈추거나 되돌릴 수 있게 만드는 신뢰 설계입니다.

FOR : PRODUCT DESIGN · FRONTEND · AX WORKFLOW · UPDATED 2026-06
AXI-01

인터랙션 설계 개요

AX 제품의 인터랙션은 사용자 의도, AI 처리 상태, 결과 검토, 승인과 복구를 하나의 흐름으로 연결합니다. 일반 UI보다 “시스템이 지금 무엇을 알고 무엇을 모르는가”를 더 명확히 드러내야 합니다.

구분일반 인터랙션AX 인터랙션
입력정해진 폼, 버튼, 메뉴자연어, 파일, 컨텍스트, 추천 명령, 반복 수정 요청
응답즉시 완료 또는 오류분석 중, 부분 결과, confidence, 검토 필요, 승인 대기
통제취소, 뒤로가기, undostop generation, rollback, diff 검토, human approval
신뢰일관된 UI와 오류 처리근거, 한계, 출처, 변경 범위, 실패 시 복구 절차
AXI-02

AX 인터랙션 원칙

원칙설계 질문나쁜 신호
예측 가능성누르면 어떤 작업이 시작되고 어디까지 바뀌는가“AI 실행” 버튼만 있고 영향 범위가 없음
통제 가능성사용자가 중단, 수정, 승인, 거절할 수 있는가생성 결과가 바로 적용됨
복구 가능성실패하거나 잘못 적용했을 때 되돌릴 수 있는가undo, version, rollback이 없음
설명 가능성AI가 왜 이 결과를 냈는지 검토 단서가 있는가근거와 입력 컨텍스트가 숨겨짐
비례성위험한 작업일수록 더 강한 확인이 있는가삭제, 권한 변경, 배포가 일반 저장과 같은 마찰
Principle

AX 인터랙션의 목표는 사용자를 놀라게 하는 것이 아니라 사용자가 시스템을 믿고 통제할 수 있게 만드는 것입니다.

AXI-03

입력과 응답 패턴

패턴쓸 때필수 피드백
명령형 입력작업 범위가 명확하고 반복 가능한 경우대상, 영향 범위, 예상 결과
자연어 입력사용자 의도가 다양하거나 탐색적인 경우해석된 의도, 부족한 정보, 수정 가능성
추천/자동완성다음 행동 후보가 제한된 경우추천 이유, 적용 전 preview, 거절 옵션
제안 수락·거절AI 결과를 사람이 검토해야 하는 경우diff, confidence, accept/reject/edit
수정 요청첫 결과가 부분적으로 맞는 경우이전 조건 유지 여부, 변경된 부분 표시
AXI-04

AI 처리 상태 디자인

AI 작업은 즉시 끝나지 않거나 부분적으로 진행될 수 있습니다. 상태 디자인은 기다림을 예쁘게 포장하는 것이 아니라 현재 작업이 살아 있는지, 멈췄는지, 사용자가 뭘 할 수 있는지 알려주는 장치입니다.

상태표현사용자 행동
대기입력 검토, queue position, 예상 시작취소, 입력 수정
분석 중현재 단계, 사용 중인 컨텍스트, 진행 단서중단, 백그라운드 전환
부분 결과streaming answer, progressive disclosure일시 중지, 추가 질문, 세부 보기
검토 필요diff, 근거, confidence, 위험 표시승인, 반려, 수정 요청
실패원인 분류, 재시도 가능 여부, 보존된 입력retry, edit, support handoff
AXI-05

마이크로인터랙션

마이크로인터랙션은 작은 피드백이지만 AX에서는 신뢰의 단서가 됩니다. 복사됨, 저장됨, 적용 대기, 자동화 중단 같은 짧은 상태가 누락되면 사용자는 AI 작업이 실제로 반영됐는지 알 수 없습니다.

요소좋은 피드백주의
버튼idle → loading → success/fail 상태가 분리됨클릭 후 아무 변화가 없으면 중복 실행을 유발
복사아이콘 변화, 짧은 “복사됨”, focus 유지toast만 띄우면 스크린리더나 빠른 흐름에서 놓침
저장저장 중, 저장됨, 저장 실패, retry자동 저장은 마지막 저장 시각과 실패 상태가 필요
적용preview, apply pending, applied, rollbackAI 결과가 즉시 적용되면 검토권을 잃음
되돌리기undo window, 변경 요약, version link위험한 변경에는 단순 toast undo만으로 부족
AXI-06

애니메이션 원칙

목적권장 사용피해야 할 사용
상태 전환 이해panel open/close, step transition, diff reveal의미 없는 bounce, 과한 delay
처리 중 피드백progress, skeleton, streaming cursor끝나지 않는 shimmer만 보여줌
공간 변화 안내새 항목 삽입, reorder, expand/collapselayout shift를 애니메이션으로 숨김
주의 환기위험 변경 confirmation, conflict highlight색과 모션만으로 위험 전달
Motion rule

AX에서 애니메이션은 “AI가 똑똑해 보이게” 만드는 장식이 아닙니다. 시스템 상태, 변경 위치, 사용자 다음 행동을 더 잘 이해하게 만들 때만 사용합니다. 기본 duration은 짧게 유지하고, 장시간 작업은 progress와 단계 설명으로 보완합니다.

AXI-07

결과 검토 인터랙션

AX 결과물은 생성보다 검토가 중요합니다. 사용자가 결과를 믿을 수 있으려면 변경 전후, 근거, 위험, 승인 상태가 같은 화면에서 연결되어야 합니다.

패턴핵심 요소검증 질문
diff 보기변경 전/후, 추가/삭제, 영향 범위사용자가 무엇이 바뀌는지 즉시 알 수 있는가
미리보기preview mode, sample data, 권한별 결과적용 전 실제 결과를 충분히 확인할 수 있는가
승인 큐요청자, 변경 요약, risk level, reviewer누가 왜 승인해야 하는지 명확한가
수정 요청comment, regenerate, keep/discard 조건기존 검토 맥락이 유지되는가
rollbackversion, 이전 상태, 영향 알림되돌린 뒤 성공 여부를 확인할 수 있는가
AXI-08

오류와 불확실성

상황표현 방식다음 행동
정보 부족부족한 입력을 구체적으로 질문필드 보완, 컨텍스트 추가
낮은 confidence불확실한 항목 표시, 근거 링크사람 검토, 비교안 선택
hallucination 위험출처 없음, 추정, 검증 필요 라벨원문 확인, 자동 적용 차단
권한 부족실행 불가 이유와 요청 경로권한 요청, 관리자 승인
위험 작업영향 범위, irreversible 여부, confirmation이중 확인, staged rollout
AXI-09

접근성과 모션 안전성

AX 인터랙션이 복잡해질수록 접근성은 더 중요합니다. 처리 상태, 스트리밍 결과, 승인 대기, 오류는 시각적 애니메이션만으로 전달하면 안 됩니다.

항목기준확인 방법
Keyboard입력, 제안 선택, 승인/반려, 중단이 키보드로 가능Tab order, Enter/Escape 동작 확인
Focus결과 생성 후 focus가 예측 가능한 위치로 이동screen reader와 키보드 흐름 테스트
Status처리 중/완료/실패가 live region 또는 명확한 text로 전달시각 효과 제거 후 의미 유지 확인
Reduced motionprefers-reduced-motion에서 불필요한 모션 제거OS 설정 변경 후 확인
Color색상과 모션만으로 상태를 전달하지 않음아이콘, 텍스트, 패턴 병행
AXI-10

실무 패턴 라이브러리

AI 입력창prompt box
placeholder보다 예시 prompt, scope selector, 첨부 컨텍스트, 제출 전 영향 범위가 중요합니다.
Prompt suggestionstarter
사용자 빈 화면을 줄이되, 추천 이유와 편집 가능성을 함께 둡니다.
Streaming answerprogressive

Generated content cardreview
결과, 근거, 출처, 수정 요청, 적용 버튼이 한 카드에서 연결되어야 합니다.
Approval queuehuman-in-the-loop
요청자, 변경 요약, 위험도, reviewer, 만료 시점을 보여줍니다.
Automation statusrun state
running, paused, waiting approval, failed, rolled back 상태를 구분합니다.
AXI-11

리뷰 체크리스트

  • 사용자가 지금 AI가 무엇을 하는지 알 수 있는가.
  • 사용자가 실행을 멈추거나 결과를 되돌릴 수 있는가.
  • AI 결과를 검토할 근거, 출처, diff, confidence가 있는가.
  • 실패했을 때 입력이 보존되고 다음 행동이 보이는가.
  • 마이크로인터랙션이 중복 실행과 불확실성을 줄이는가.
  • 애니메이션이 상태 이해를 돕는가, 아니면 장식으로 신뢰를 흐리는가.
  • prefers-reduced-motion, keyboard, focus, live status가 반영됐는가.
AXI-12

실전 사례

사례핵심 인터랙션품질 기준
AI 문서 요약원문 출처, 요약 confidence, 원문 대조출처 없는 문장은 자동 확정하지 않음
디자인 시안 생성variant 비교, prompt history, 선택 이유선택된 안과 버려진 안의 기준이 기록됨
업무 자동화 실행run state, stop, retry, approval checkpoint장시간 작업이 방치되지 않고 상태가 관측됨
관리자 승인 플로우risk level, diff, reviewer comment, audit log승인자가 변경 영향과 책임을 이해함
위험 작업 confirmation대상, irreversible 여부, typed confirmation실수 방지 마찰이 위험도에 비례함
AX INTERACTION REVIEW Intent: 사용자가 하려는 일이 무엇인가
State: AI가 지금 무엇을 처리 중인가
Evidence: 결과를 검토할 근거가 있는가
Control: 중단, 수정, 승인, rollback이 가능한가
Motion: 애니메이션이 의미를 전달하는가
