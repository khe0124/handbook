LCA
상세 화면·기능 목록 CRUD 분해

상세 목록을 엔티티별 오퍼레이션(CRUD) 단위로 더 쪼갰습니다. 모델 노드·배경 데이터셋·결과 스냅샷마다 생성·조회·수정·삭제 + 도메인 오퍼레이션(import·calculate·lock·verify)이 따로 존재합니다. 특히 게시된 결과·고정된 DB 버전은 수정·삭제가 금지라, CRUD로 분해해야 재현성을 떠받치는 제약이 드러납니다.

LCA의 재현성은 버전 고정에서 옵니다. BackgroundDataset 바인딩·LCIAMethod는 게시 시 LK로 잠기고, Result 스냅샷은 U·D가 금지(재계산은 새 버전), 비판적 검토 의견은 패널만 C하고 분석가는 변경 불가입니다. 또 공개 비교 주장은 게시(LK) 전에 동등성·패널 검토 게이트를 통과해야 합니다. 아래 상태·제약 컬럼이 이 "할 수 없음"과 게이트를 행마다 명시합니다.
