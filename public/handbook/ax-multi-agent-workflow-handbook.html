Multi-Agent Workflow

멀티 에이전트 워크플로우는 많은 에이전트를 돌리는 기술이 아니라 서로 다른 책임과 관점을 분리해 자기검열의 약점을 줄이는 운영 방식입니다.

MA-01

역할 분리

코드를 작성한 에이전트에게 최종 평가까지 맡기지 않습니다. 작성자는 구현 의도에 끌리고, 리뷰어는 실제 동작과 위험에 집중해야 합니다. independent roles reduce risk: 독립 역할은 같은 착각을 반복하는 자기검열 위험을 낮춥니다.

역할책임출력
Planner문제를 쪼개고 범위와 성공 조건 정의task breakdown
Explorer코드베이스와 원인 후보 조사facts, relevant files, hypotheses
Implementer제한된 범위 안에서 구현patch, notes, tests
Verifier테스트, 타입체크, 실행 결과 확인verification report
Reviewer작성자와 독립적으로 위험과 누락 탐지findings, residual risk
Integrator결과 통합, 충돌 제거, 최종 판단merged change, final rationale
MA-02

언제 쓰는가

적합부적합
인증, 결제, 권한, 데이터 마이그레이션처럼 실패 비용이 큰 변경한 파일의 작은 문구 수정
여러 접근법을 비교해야 하는 설계 판단요구사항이 불명확한 상태
프론트엔드, 백엔드, DB, 테스트 작업이 독립적으로 나뉨검증 기준이 없는 작업
작성과 리뷰를 분리해야 하는 고위험 변경통합 비용이 작업 자체보다 큰 경우
MA-03

워크트리 전략

여러 에이전트가 같은 브랜치에서 동시에 수정하면 충돌과 책임 혼란이 생깁니다. 독립 워크트리는 실험, 구현, 리뷰, 회귀 확인을 분리합니다.

WORKTREE LAYOUT main
feature/auth-session-fix
worktree/a-repro-test
worktree/b-cookie-policy-fix
worktree/c-review
  1. 기준 브랜치를 고정합니다.
  2. 각 에이전트에게 독립 워크트리를 줍니다.
  3. 각 워크트리는 하나의 목표만 가집니다.
  4. 결과는 patch, test result, risk memo로 제출합니다.
  5. 통합자는 하나의 최종 변경만 선택하거나 병합합니다.
worktree isolation

worktree isolation은 에이전트별 파일 소유권, 검증 로그, 폐기 가능한 실험 공간을 분리해 통합자가 원인과 책임을 추적하게 합니다.

MA-04

도구 조합

도구언제 쓰나주의점
Skill반복 가능한 절차가 중요할 때절차가 낡으면 계속 같은 실수 반복
Plugin특정 도구 묶음과 워크플로우가 필요할 때권한과 side effect 확인
ConnectorGitHub, Jira, DB, 이메일 등 외부 데이터가 필요할 때외부 콘텐츠는 untrusted로 취급
Subagent독립적인 분석·구현·리뷰가 필요할 때입력 범위와 출력 형식을 좁힘
Rule

반복 절차가 중요하면 skill, 외부 도구 실행이 필요하면 plugin, 외부 데이터가 필요하면 connector, 독립 판단이 필요하면 subagent를 씁니다.

MA-05

통합과 리뷰

병렬 작업의 성패는 에이전트 수가 아니라 integration checklist와 conflict-handling 기준에 달려 있습니다. 각 결과에 검증 증거가 없으면 통합하지 않습니다.

INDEPENDENT REVIEW PROMPT 너는 이 변경을 작성하지 않은 독립 리뷰어다.
목표는 칭찬이나 요약이 아니라 결함 탐지다.
1. 실제 버그 또는 회귀 가능성
2. 보안, 인증, 권한, 데이터 손실 위험
3. 누락된 테스트
4. 요구사항과 구현의 불일치
5. 과도한 변경 또는 불필요한 추상화
parallel review exercise: 같은 diff를 Reviewer와 Verifier가 독립 검토한 뒤 finding 차이를 비교
실패대응
모든 에이전트가 같은 가정을 공유역할별 입력과 질문을 다르게 설계
subagent 결과를 검증 없이 합침검증 증거 없는 결과는 통합 보류
여러 에이전트를 돌려 비용만 증가병렬화 가치와 통합자를 사전에 지정
워크트리 없이 같은 파일 동시 수정파일 소유권과 write scope 분리
