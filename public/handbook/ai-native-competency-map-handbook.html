AI Native 역량
매트릭스와 진단

AI Native 역량은 "AI를 써봤다"가 아니라 제품 기능, 평가, 보안, 운영, 작업 루프를 증거로 닫는 능력입니다.

OUTPUT : competency scorecard · evidence ledger · next training queue
MAP-01

판정 기준

역량 진단은 감상이나 자신감이 아니라 제출 가능한 산출물로 판정합니다. 설명할 수 있어도 재현 가능한 증거가 없으면 아직 실무 역량으로 보지 않습니다.

수준판정 기준대표 증거
L1 입문개념을 설명하고 작은 예제를 따라 할 수 있다학습 메모, 작은 실습 결과
L2 실무 가능제한된 기능을 구현하고 기본 검증을 남긴다테스트, prompt contract, API 계약
L3 독립 운영 가능실패 모드, 보안 경계, 운영 지표까지 책임진다eval report, runbook, incident packet
L4 리드 가능팀 표준, 리뷰 기준, 자동화, 거버넌스를 설계한다rubric, CI gate, approval policy, dashboard
MAP-02

AI Native 역량 축

역량핵심 질문실패 신호증거
소프트웨어 기본기AI가 만든 코드를 검증할 수 있는가실행은 되지만 왜 맞는지 설명하지 못함테스트, 타입, 디버깅 노트
제품 문제 정의사용자 문제와 비목표가 분리됐는가데모 기능은 있으나 성공 기준이 없음problem brief, user flow
LLM 설계prompt, schema, fallback이 계약으로 남는가프롬프트 문장만 있고 출력 계약이 없음prompt contract, JSON schema
RAG 품질검색 실패와 생성 실패를 분리하는가오답을 프롬프트만 고쳐 해결하려 함retrieval eval report
평가 하네스변경 전후 품질 하락을 잡는가샘플 몇 개로 품질을 판단함eval dataset, CI gate
보안injection, 권한, 데이터 유출을 재현하는가정책 문서만 있고 공격 fixture가 없음red-team report
Agent Runtime도구 호출을 승인, 감사, 재생할 수 있는가에이전트가 실행한 일을 사후 추적하지 못함approval ledger, replay trace
운영비용, 지연, 품질, fallback을 관측하는가장애 때 프롬프트 수정부터 시도함dashboard spec, LLM runbook
AX 루프AI와 협업한 과정을 재현 가능한 에피소드로 남기는가AI 산출물을 사람이 검증한 기록이 없음context package, episode log
증거화경험을 과장 없이 포트폴리오로 설명하는가"AI 활용"만 말하고 지표와 실패가 없음evidence pack
MAP-03

빠른 수준 진단

각 질문에 대해 현재 남길 수 있는 증거가 없다면 해당 수준은 아직 통과하지 않은 것으로 봅니다.

질문L1L2L3L4
LLM 품질을 어떻게 판단하는가샘플 답변을 본다golden set을 만든다회귀 gate를 둔다팀 릴리스 기준으로 운영한다
RAG 오답을 어떻게 고치는가프롬프트를 바꾼다retrieval과 generation을 분리한다chunk/rerank/citation 지표를 본다index 운영과 drift를 관리한다
도구 호출을 어떻게 통제하는가schema를 정의한다approval을 둔다idempotency와 audit을 둔다policy와 replay를 표준화한다
AI 경험을 어떻게 설명하는가사용 도구를 말한다구현 기능을 말한다평가/보안/운영 증거를 말한다팀 표준화 경험을 말한다

MAP-05

보강 루프

COMPETENCY HARDENING LOOP 1. 현재 수준을 산출물 기준으로 판정한다
2. 가장 낮은 역량 축 하나를 고른다
3. AI Native 실습 랩에서 해당 fixture를 실행한다
4. 실패 분석과 개선 diff를 남긴다
5. evidence ledger를 업데이트한다
6. 포트폴리오 문장으로 바꿔 설명한다
완료 기준

이 문서를 끝냈다는 뜻은 표를 읽었다는 뜻이 아니라, 현재 자신의 L1-L4 위치와 다음 실습 랩, 제출할 산출물을 정했다는 뜻입니다.
