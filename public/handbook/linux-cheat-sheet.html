LX-CS-01

개요

운영 조사에서는 명령을 많이 아는 것보다 현재 위치, 대상 파일, 프로세스, 포트, 로그 범위를 안전하게 좁히는 것이 중요합니다.

상황먼저 확인할 것
파일 찾기pwd, ls -la, find, rg
포트 점유lsof -i :3000, ss -tlnp
로그 확인tail -f, less, grep, journalctl
리소스 확인df -h, du -sh *, top, free -h
LX-CS-02

핵심 명령/개념

명령압축 설명
rg "text"현재 디렉터리 이하에서 빠른 텍스트 검색
tail -f app.log로그를 실시간으로 따라감
ps aux | grep node프로세스 목록에서 대상 확인
kill PIDSIGTERM으로 정상 종료 요청
chmod +x file실행 권한 부여
du -sh * | sort -rh용량 큰 항목을 정렬
LX-CS-03

판단 기준

선택기준
cat작은 파일 전체를 즉시 볼 때
less큰 파일을 검색하며 볼 때
tail -f새로 쌓이는 로그를 실시간 확인할 때
kill정상 종료 요청이 가능할 때
kill -9정상 종료가 실패한 마지막 수단
LX-CS-04

자주 나는 실수

  • rm -rf 전에 대상 경로를 확인하지 않는다.
  • 큰 로그를 cat으로 터미널에 모두 쏟는다.
  • kill -9를 먼저 써서 graceful shutdown 기회를 없앤다.
  • 권한 문제를 무조건 sudo로 덮고 원인을 보지 않는다.
  • macOS BSD 명령과 GNU/Linux 옵션 차이를 고려하지 않는다.
LX-CS-05

면접/리뷰용 한 줄 답변

파이프pipe
셸의 힘은 작은 명령의 표준 출력을 다음 명령의 표준 입력으로 연결해 조합하는 데 있습니다.
권한permission
Unix 권한은 소유자, 그룹, 기타 사용자에게 읽기, 쓰기, 실행 권한을 나눠 부여하는 모델입니다.
프로세스 종료signal
먼저 SIGTERM으로 정리할 시간을 주고, 실패할 때만 SIGKILL을 사용합니다.
