형상 관리와
변경 추적

형상 관리는 Git 저장소만 뜻하지 않습니다. 운영에 영향을 주는 코드, 설정, 인프라 정의, 파이프라인, 아티팩트 버전을 추적 가능한 상태로 관리하는 체계입니다.

CFG-01

형상 관리 범위

대상예시관리 이유
Application codesrc, tests기능 변경 추적
Configurationyaml, env schema환경 차이 통제
InfrastructureTerraform, Helm chart운영 리소스 변경 추적
PipelineGitHub Actions, GitLab CI검증·배포 절차 버전 관리
PolicyRBAC, OPA, branch rule권한과 통제 변경 추적
CFG-02

버전과 추적성

운영 장애가 났을 때 “지금 무엇이 배포되어 있는가”를 답할 수 있어야 합니다.

TRACE CHAIN Issue ID → Pull Request → Commit SHA → Build ID → Artifact Digest → Release Tag → Deployment Record
식별자쓰임
Commit SHA코드 상태 고정
Build ID파이프라인 실행 추적
Image digest컨테이너 이미지 불변 식별
Release tag사용자에게 나간 기준선
Deployment ID환경 반영 기록
CFG-03

리뷰 흐름

코드 리뷰는 취향 토론이 아니라 변경 위험을 낮추는 구조적 장치입니다.

리뷰 관점질문
의도왜 이 변경이 필요한가
정확성요구와 구현이 맞는가
운영 영향배포, 데이터, 성능, 보안 영향은 무엇인가
테스트실패해야 할 경우를 잡는가
복구문제가 생기면 되돌릴 수 있는가
실무 감각

PR 크기가 커질수록 리뷰 품질은 떨어집니다. 좋은 DevOps 체계는 큰 변경을 작은 PR과 feature flag, migration 단계로 쪼갭니다.

CFG-04

릴리스 기준선

릴리스 태그는 “이 시점의 코드와 아티팩트가 운영 반영 후보”라는 기준선입니다.

항목기준
Tagv1.8.0, immutable 원칙
Release note기능, 수정, migration, 위험, rollback
Artifacttag와 image digest 연결
Approval누가 어떤 근거로 승인했는지 기록
CFG-05

권한 통제

Protected branchmerge control
main/release branch는 직접 push를 막고 PR, CI 통과, review 승인 후 merge합니다.
Least privilege권한 최소화
배포 토큰, registry 권한, secret 접근은 필요한 범위와 기간으로 제한합니다.
Audit log감사
누가 branch rule, secret, pipeline 권한을 바꿨는지 추적할 수 있어야 합니다.
Break-glass비상 권한
장애 대응용 예외 권한은 사용 사유, 시간, 사후 리뷰를 남깁니다.
