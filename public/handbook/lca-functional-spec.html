LCA
UI/UX 기능정의서

제품의 환경영향을 요람에서 무덤까지 모델링·계산·해석하는 과정을 소프트웨어로 옮기면 어떤 화면이 필요한가. LCA 도메인 핸드북의 방법론(기능단위·경계·배분·LCIA·불확실성)을 화면 인벤토리와 화면별 기능정의로 번역한 기획 문서입니다. 일반 B2B SaaS(제조·컨설팅·지속가능성 팀용)를 가정합니다.

SCOPE : 제품 단위 LCA 모델링·계산·해석·발자국 산출·검토 SaaS
ASSUMPTION : 특정 고객사·특정 플랫폼 비종속 — 도메인 표준 기반 일반 기획안
COMPANION : LCA 도메인 핸드북 · 탄소회계 기능정의서
FS-00

문서 개요·전제

LCA 도구의 본질은 선형대수 솔버 + 데이터베이스 + 해석 워크벤치입니다. 무엇을 만들고 무엇은 다루지 않는지를 먼저 못 박습니다.

항목내용
목적제품 시스템을 모델링하고 전과정 영향을 계산·해석·보고하는 제품의 화면·기능 정의
대상 제품LCA 분석가·제조 지속가능성 팀이 쓰는 B2B SaaS. 다(多)연구·재현 가능 워크플로 전제
준거 표준ISO 14040/14044(LCA) · ISO 14067(CFP) · ISO 14025(EPD/PCR) · LCIA 방법론(EF 3.1 등)
포함목적·범위 · 제품시스템 모델링 · LCI · 배경DB · 배분 · LCIA · 결과·민감도·불확실성 · CFP/EPD · 비판적 검토
제외특정 배경 DB 라이선스 계약 · 특정 PCR 전문(全文) 내장 · 고객별 보고 서식. 단, 외부 사용자(패널·구매자)·상용 DB를 다루므로 보안·테넌트 격리·라이선스 가드는 제외가 아니라 FS-14에서 정의
전제·가정모든 화면·기본값은 도메인 표준 기반 제안. 방법론 선택(기인/결과, 배분방식, LCIA 패키지)은 연구 목적에 종속하므로 설정으로 받음
기획자 노트 — 추측하지 않은 것

배분 방식 기본값, LCIA 방법론 기본값, 컷오프 임계, 자본재 포함 여부 같은 항목은 연구 목적·표준·고객에 따라 갈리는 방법론 선택이라 임의로 고정하지 않았습니다. 화면은 이를 명시적으로 선택·기록하게 설계 — LCA의 신뢰는 "무엇을 골랐는지 드러내는 것"에서 나오기 때문입니다.

FS-01

페르소나·역할

LCA는 모델 구축 → 계산 → 해석 → 검토의 분업이며, 공개 비교 주장에는 외부 패널 검토가 의무입니다. 결과 소비자(제품팀)와 모델 작성자의 니즈가 크게 다릅니다.

PRIMARY · ANALYST
LCA 분석가

제품 시스템을 모델링하고 배경 데이터를 연결, 계산을 돌린다. 단위공정 모델러와 데이터 매핑에서 가장 오래 머문다.

REVIEWER · INTERNAL
내부 도메인 검토자

가정·경계·배분의 타당성과 데이터 품질을 점검한다. 민감도·기여도 요약을 본다.

CRITICAL REVIEW · EXTERNAL
비판적 검토 패널 (3인)

공개 비교 주장의 ISO 14044 의무 검토. 가정·방법·결론을 독립 검증하고 의견·대응을 남긴다. 읽기·코멘트 중심.

DATA STEWARD
데이터 관리자

배경 DB(ecoinvent 등) 버전·라이선스, 사내 단위공정 라이브러리, 계수를 관리한다.

DECISION MAKER
제품·의사결정자

결과를 소비한다(어느 설계가 나은가). 핫스팟·비교·불확실성 요약만 본다 — 모델 내부는 안 본다.

EXCHANGE · EXTERNAL
구매자/공급사 (PCF 교환)

완성된 제품발자국(PCF)을 받거나 보낸다. 게시된 결과만 제한 접근.

설계 함의

분석가는 모델 내부(단위공정·플로우)를, 의사결정자는 결과 표면(핫스팟·비교)을 봅니다. 같은 연구를 두 층위의 뷰로 제공해야 합니다 — 모델 워크벤치 vs 결과 리포트. 비판적 검토 패널은 둘 다 읽되 변경은 못 하고 코멘트만(→ FS-07).

FS-02

정보구조(IA)·화면맵

ISO 14040의 4단계(목적·범위 → LCI → LCIA → 해석)가 그대로 제품의 골격이 됩니다. 여섯 영역으로 펼칩니다.

① 연구 관리

  • 프로젝트 대시보드 연구 목록·상태
  • 목적·범위 정의 기능단위·경계·컷오프

② 모델링 (LCI)

  • 제품시스템 모델러 단위공정 그래프
  • LCI 데이터 입력 전경 1차 데이터
  • 배경 DB 연결 ecoinvent·EF 매핑
  • 데이터 품질 페디그리

③ 계산 설정

  • 배분 설정 위계·재활용·CFF
  • LCIA 방법론 EF3.1·GWP·중간/종말

④ 결과·해석

  • 결과·기여도 핫스팟·단계분해
  • 민감도·시나리오 파라미터·전력망
  • 불확실성 몬테카를로·보수적
  • 비교 분석 기능단위 동등성

⑤ 산출·준수

  • CFP / ISO 14067 탄소발자국
  • EPD / PCR 제품군 규칙 준수
  • PCF 게시·교환 → Scope 3

⑥ 거버넌스

  • 비판적 검토 3인 패널 워크플로
  • 버전·재현성 모델·DB 스냅샷
  • 사용자·권한 RBAC

모델링(②)과 계산 설정(③)은 한 연구의 워크벤치 안에서 연결된 탭으로 동작하고, 결과(④)는 소형 모델이면 동기 재계산, 대형 시스템·몬테카를로는 비동기 job(진행률 표시, → FS-09.3·FS-15)으로 갱신됩니다 — 4단계가 폭포수가 아니라 반복(iterative)이라는 ISO 원칙(LCA-01)을 UI가 반영합니다.

FS-04

핵심 화면 기능정의

P0·P1 화면을 기능정의 카드로 상술합니다. 목적 · 핵심 기능 · 데이터/상태/검증 · 핸드북 연계.

상술 범위 — 솔직한 표기

이 장은 전 화면이 아니라 선별 상술입니다. 카드가 없는 화면(SC-01·04 LCI·06·10·12 해석·14 CFP·15·18 버전/RBAC)도 FS-03 인벤토리에 정의돼 있으며, 잔여 P0 카드(특히 SC-04 LCI 입력·SC-18 버전·재현성/RBAC, flagship 출력 SC-14 CFP)는 상세설계 단계에서 완성합니다. 수용 기준은 FS-16, 빠진 도메인 개념은 FS-13으로 보강했습니다.

SC-07배분 설정P0
LCA 최대의 방법론 논쟁을 화면으로. 다산출 공정마다 ISO 위계를 따르도록 강제하고, 선택을 기록하며, 대안 방식의 민감도를 함께 보게 한다.
핵심 기능
  • 다산출 공정 자동 식별, ISO 14044 위계 안내: ①배분 회피(세분화 / system expansion) → ②물리적 배분 → ③경제적 배분. attributional allocation과 consequential substitution을 구분
  • system expansion·대체(substitution) 시 avoided-burden(음수 flow) 시각화, CFF 파라미터(A·R·Qsin/Qsout) 노출
  • 재활용 배분 접근 선택(recycled-content / end-of-life / CFF)
  • 시스템 전체 부담 보존 검사(증발·복제 차단)
  • 대안 배분 방식 민감도 자동 생성(→ SC-10)
데이터

데이터 모델 개요

화면 뒤의 핵심 엔티티. LCA 모델은 본질적으로 그래프(단위공정·플로우) + 버전 고정된 배경 데이터입니다.

CORE INSIGHT — 재현성은 버전 고정에서

BackgroundDataset·LCIAMethod·Result는 불변 버전 레코드입니다. ecoinvent가 갱신되거나 AR 버전이 바뀌어도 과거 연구는 자기가 쓴 버전을 영구 참조 — "정확히 어떤 버전의 어떤 산식으로 계산됐는가"(LCA-07)를 데이터 구조가 보장합니다. 탄소회계 기능정의서의 계수 버전 고정과 동일한 설계.
