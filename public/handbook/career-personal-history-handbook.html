FRONTEND ANSWER CARD 저는 디자인 기반에서 출발해 React/Next/TypeScript 중심의 B2B·SaaS 프론트엔드로 성장했습니다. 특히 LCA, pople, TaaS처럼 도메인 규칙이 복잡한 서비스에서 화면, 폼, 테이블, 권한, 상태 전이, 리포트, i18n을 구현해 왔습니다. 단순히 화면을 만드는 것보다 사용자가 어떤 업무를 완료해야 하는지, 어떤 입력 오류를 회복해야 하는지, 어떤 API 계약에 의존하는지를 먼저 정리하는 편입니다.
B2B SAAS ANSWER CARD LCA에서는 공정별 데이터 입력, LCI DB, 공정 흐름도, 권한관리, OCR 활동데이터 연동까지 다뤘고, pople에서는 레지스트리 공개 웹, 거래 마켓, 어드민을 같은 도메인 안에서 구현했습니다. B2B·SaaS 화면은 예쁜 UI보다 업무 흐름, 권한, 데이터 검증, 반복 작업 효율이 중요하다고 봅니다. 저는 복잡한 요구사항을 사용자가 따라갈 수 있는 화면 구조와 상태로 바꾸는 데 강점이 있습니다.
LCA ANSWER CARD LCA 프로젝트에서는 계산 산식 자체를 소유했다기보다, 복잡한 전과정평가 업무를 사용자가 입력하고 검증하고 결과를 확인할 수 있는 제품 흐름으로 구현했습니다. 공정별 데이터 입력, 엑셀 업로드, LCI DB 연동, 공정도 에디터, 권한관리, OCR 결과 편집까지 다루면서 복잡한 도메인을 UI와 상태 모델로 분해하는 경험을 쌓았습니다.
TAAS ANSWER CARD TaaS에서는 암호화폐 세금 신고 서비스의 프론트엔드를 단독으로 구축했습니다. 거래소 API Key 등록, MetaMask 지갑 연동, 거래내역 조회와 필터, 손익·예상 세액 표시, 한·영·일 다국어, Amplitude 분석, 어드민 RBAC와 유저 상세까지 담당했습니다. 세금 산식 자체는 서버와 도메인 정책에 의존했고, 저는 사용자가 결과를 이해하고 검증할 수 있는 화면과 운영 콘솔을 구현했습니다.
HEMS ANSWER CARD HEMS에서는 React Native 기반으로 EV 충전기, 태양광, 온수기, 세탁건조기, HVAC 같은 에너지 기기 상태를 대시보드와 위젯으로 표현했습니다. Redux Saga로 기기·알림·스케줄·TOU 요금 상태를 다뤘고, 타임존과 알림 이동 같은 실제 사용 조건을 반영했습니다. 웹 중심 경력에 모바일 IoT 경험을 더해주는 증거로 사용할 수 있습니다.
AI NATIVE ANSWER CARD AI를 단순 코드 생성 도구로 쓰기보다, 요구사항 정리, 문서 구조화, 테스트 기준, 리뷰 체크리스트를 만드는 데 활용하고 있습니다. 중요한 것은 AI가 만든 결과를 그대로 믿는 것이 아니라, 테스트와 빌드, 릴리스 기준으로 검증 가능한 산출물로 바꾸는 것입니다.
