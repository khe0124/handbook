리눅스 / 셸 명령어
핸드북

명령어 암기가 아니라 사고 모델을 먼저 잡습니다 — 모든 명령은 텍스트 스트림을 입력받아 텍스트 스트림을 내보내고, 작은 도구들을 파이프로 조립해 큰 일을 합니다. 이 한 가지 철학을 이해하면 나머지는 조합입니다. 네 환경(macOS·zsh·mise·chezmoi)과 모던 CLI(rg·fd·bat) 기준.

NOTE : macOS는 BSD 계열 — 일부 플래그가 GNU(Linux)와 다름. 차이 나는 곳은 표기함
COMPILED : 2026-06
LX-01

셸의 사고 모델

Unix 철학 한 줄: "한 가지를 잘하는 작은 프로그램을, 텍스트 스트림으로 잇는다." 명령어를 외우기 전에 이 구조를 잡으면, 처음 보는 명령도 같은 문법으로 읽힙니다.

ANATOMY OF A COMMAND command -옵션 --긴옵션 인자(파일·경로)
예: ls -la ~/projects

THREE STREAMS stdin(0) ──▶ [ 프로그램 ] ──▶ stdout(1)
└──▶ stderr(2) ← 에러는 별도 채널

모든 프로세스엔 세 개의 표준 스트림이 있습니다 — 입력(stdin), 정상 출력(stdout), 에러 출력(stderr). 이게 분리돼 있어서 정상 결과는 파일로 저장하면서 에러는 화면에 띄우는 식의 조합이 가능합니다(→ LX-08). 그리고 모든 명령은 끝나면 종료코드(exit code)를 남깁니다 — 0은 성공, 그 외는 실패. echo $?로 직전 명령의 종료코드를 보고, &&/||로 성공·실패에 따라 다음을 잇습니다.

도움 받기의미
man ls매뉴얼 페이지 (q로 종료, /단어로 검색)
ls --help간단 사용법 (대부분의 GNU 도구)
tldr ls예제 중심 요약 (별도 설치 — 실전에 가장 빠름)
which ls · type ls이 명령이 어디 있는지 / alias·함수인지
LX-02

탐색·이동

디렉터리 트리 안에서 "지금 어디고, 뭐가 있고, 어디로 갈까". 가장 자주 치는 명령들이라 손에 붙여둘 가치가 큽니다.

명령의미 / 자주 쓰는 형태
pwd현재 경로 출력
ls -la숨김파일 포함(-a) 상세(-l). -lh 사람이 읽는 크기, -lt 시간순, -ltr 오래된 순
cd -직전 디렉터리로 토글. cd 단독은 홈(~)으로, cd .. 상위
tree -L 2트리 구조 2단계까지 (별도 설치). -d 디렉터리만
find . -name "*.ts"이름으로 찾기. -type f/d(파일/디렉터리), -mtime -7(7일 내 수정)
fd 패턴find의 모던 대체 — 빠르고 직관적, .gitignore 자동 존중. fd -e ts(확장자)
경로 기본기

.=현재 · ..=상위 · ~=홈 · /=루트 · -=직전(cd 한정). 절대경로(/Users/...)는 어디서 쳐도 같은 곳, 상대경로(./src)는 현재 위치 기준. 스크립트에선 위치에 안 흔들리게 절대경로가 안전합니다.

LX-03

파일 보기·편집

"내용만 잠깐 볼 것"과 "고칠 것"은 도구가 다릅니다. 큰 로그를 cat으로 통째로 쏟지 마세요.

명령언제 / 형태
cat file전체를 한 번에 출력 — 작은 파일에만. 여러 파일 이어붙이기에도 씀
less file페이지 단위로 큰 파일 탐색. /검색, G(끝), g(처음), q(종료)
head -n 20 / tail -n 20앞/뒤 N줄. 구조·결과 빠른 확인
tail -f app.log실시간 추적 — 로그가 쌓이는 걸 계속 따라감. 운영 디버깅 필수
bat filecat의 모던판 — 문법 강조·줄번호·페이징 (별도 설치)
vim file터미널 편집기. 최소 생존: i(입력)→Esc→:wq(저장종료) / :q!(버리고 종료)
실전 팁

에러를 쫓을 땐 tail -f에 필터를 얹습니다: tail -f app.log | grep ERROR — 새로 찍히는 에러만 실시간으로. nano가 vim보다 편하면 nano도 충분합니다(저장 Ctrl-O, 종료 Ctrl-X). 편집기 전쟁에 끼지 말고 탈출하는 법만 확실히 익히세요.

LX-04

검색 — 텍스트와 파일

두 종류의 검색을 구분하세요 — 파일 안의 내용을 찾는가(grep/rg), 파일 자체를 찾는가(find/fd). 개발 일상에서 가장 자주 쓰는 무기입니다.

명령의미 / 형태
grep "패턴" file파일 안에서 줄 찾기. -i(대소문자 무시) -n(줄번호) -r(재귀) -v(반대=제외) -C 3(앞뒤 3줄)
rg "패턴"ripgrep — grep의 모던 대체. 빠르고 .gitignore 자동 존중, 재귀가 기본. rg -t ts "useState"(타입 한정)
rg -l "패턴"매칭된 파일 이름만 나열 (-l)
find . -name "*.log"이름 패턴으로 파일 찾기
fd -e log확장자로 (모던, 빠름)
find . -name "*.tmp" -delete찾아서 삭제 — 실행 전 -delete 빼고 먼저 확인
grep과 정규식

grep의 패턴은 정규식입니다 — .(아무 글자) *(0회 이상) ^(줄 시작) $(줄 끝). 특수문자를 글자 그대로 찾으려면 이스케이프하거나 grep -F(고정 문자열). 코드베이스 탐색은 거의 항상 rg가 정답입니다 — 빠르고, 빌드 산출물·node_modules를 알아서 건너뜁니다.

LX-05

파일 조작

복사·이동·삭제·생성. 단순하지만 rm은 되돌릴 수 없으니, 파괴적 명령엔 손이 한 번 멈춰야 합니다.

명령의미 / 형태
cp src dst복사. 디렉터리는 -r(재귀). -i 덮어쓰기 전 확인, -p 속성 보존
mv src dst이동 겸 이름 변경 (같은 위치로 옮기면 rename)
rm file삭제. -r 디렉터리, -f 강제(확인 없음), -i 하나씩 확인
mkdir -p a/b/c디렉터리 생성. -p 중간 경로까지 한 번에
touch file빈 파일 생성 / 기존 파일 수정시각 갱신
ln -s 대상 링크심볼릭 링크(바로가기) 생성. -s 없으면 하드링크
절대 하지 말 것

rm -rf / 또는 변수 미설정 상태의 rm -rf $VAR/($VAR이 비면 rm -rf /가 됨)는 시스템을 지웁니다. 안전 습관: ① rm 전에 같은 인자로 ls를 먼저 쳐서 대상을 눈으로 확인 ② 와일드카드(*) 삭제는 특히 조심 ③ 중요한 건 rm 대신 휴지통으로 보내는 도구(trash)를 쓰는 것도 방법.

LX-06

권한·소유권

Unix 권한은 외울 게 아니라 모델입니다 — 누구(소유자/그룹/기타)가 무엇(읽기/쓰기/실행)을 할 수 있는가. ls -l의 맨 앞 10글자가 그 지도입니다.

ls -l 의 권한 비트 -rwxr-xr--   1 haeun staff  ...
│ └user└group└others
type   소유자=읽기·쓰기·실행 그룹=읽기·실행 기타=읽기

숫자(8진수) 표기 r=4   w=2   x=1  →   rwx=7   r-x=5   rw-=6
예: chmod 755 = rwxr-xr-x  |  chmod 644 = rw-r--r--
명령의미 / 형태
chmod 755 file권한을 8진수로 설정 (스크립트 실행 권한엔 보통 +x)
chmod +x script.sh실행 권한 추가 (기호 표기). -x 제거
chown user:group file소유자·그룹 변경 (보통 sudo 필요)
umask새로 만드는 파일의 기본 권한 마스크 확인/설정
sudo 명령관리자 권한으로 실행 — 무엇을 왜 올리는지 알고 쓸 것
실전

"Permission denied"의 90%는 두 가지입니다 — ① 스크립트에 실행 권한이 없음(chmod +x) ② 남의 소유 파일·시스템 경로를 건드림(sudo 필요하거나, 애초에 건드리면 안 됨). 755(실행파일·디렉터리)와 644(일반 파일)만 손에 익혀도 일상은 충분합니다.

LX-07

프로세스·잡

"포트가 이미 쓰이고 있어요", "멈춘 프로세스를 죽여야 해요" — 백엔드 일상의 단골입니다. 프로세스를 보고, 신호를 보내고, 백그라운드로 다루는 법.

명령의미 / 형태
ps aux | grep node실행 중 프로세스에서 node 찾기 (PID 확인)
top / htop실시간 자원 모니터 (CPU·메모리). htop이 보기 편함(별도 설치)
kill PID프로세스에 종료 신호(SIGTERM) — 정상 종료 요청
kill -9 PIDSIGKILL — 강제 종료. 마지막 수단(정리 못 하고 죽음)
lsof -i :30003000 포트를 누가 쓰는지 확인 → PID 찾아 kill
명령 &백그라운드로 실행
jobs / fg / bg백그라운드 잡 목록 / 앞으로 / 뒤로. Ctrl-Z로 일시정지 후 bg
nohup 명령 &터미널 닫혀도 계속 실행 (장시간 작업)
SIGNAL — kill은 "죽이기"가 아니라 "신호 보내기"

kill은 프로세스에 신호를 보내는 명령입니다. 기본 SIGTERM(15)은 "정리하고 끝내라"는 정중한 요청이라 프로세스가 받아 graceful shutdown을 합니다. SIGKILL(9)은 OS가 즉시 죽여 거부할 수 없지만, 열린 파일·연결을 못 정리합니다. 먼저 kill PID, 안 죽으면 kill -9 순서가 맞습니다. Ctrl-C는 포그라운드 프로세스에 SIGINT를 보내는 것입니다.

LX-08

파이프·텍스트 가공

셸의 진짜 힘이 여기 있습니다. 작은 도구를 |로 이어 데이터를 흐르게 하면, 한 줄로 로그 분석·집계가 됩니다. "조립"의 영역입니다.

기호의미
cmd1 | cmd2cmd1의 출력을 cmd2의 입력으로 (파이프)
cmd > file출력을 파일로 (덮어쓰기). >>는 이어쓰기
cmd 2> err.log에러(stderr)만 파일로. 2>&1은 에러를 출력에 합침
cmd > out 2>&1출력과 에러를 모두 한 파일로
도구의미 / 형태
wc -l줄 수 세기 (-w 단어, -c 바이트)
sort / sort -u정렬 / 정렬+중복제거. -n 숫자, -r 역순
uniq -c연속 중복을 세기 (정렬 후 써야 함)
cut -d',' -f1구분자(-d)로 잘라 N번째 필드(-f). CSV 다루기
tr 'a-z' 'A-Z'문자 치환·삭제 (대문자화 등)
xargs앞 출력을 다음 명령의 인자로 변환. fd -e log | xargs rm
sed 's/old/new/g'스트림 치환 (find&replace). 파일 직접 수정은 -i
awk '{print $2}'열 단위 가공 — 공백 구분 2번째 열 출력 등. 가벼운 표 처리
WORKED EXAMPLE — 접속로그에서 상위 IP 5개 cat access.log | awk '{print $1}' | sort | uniq -c | sort -rn | head -5
└ IP 추출 → 정렬 → 개수 세기 → 많은 순 정렬 → 상위 5 — 작은 도구 5개의 조립
macOS 주의 — sed -i

macOS(BSD)의 sed -i는 백업 확장자 인자를 요구합니다 — sed -i '' 's/a/b/' file(빈 따옴표 필수). GNU(Linux)는 sed -i 's/a/b/' file. 스크립트 이식할 때 단골 함정이라, 헷갈리면 brew install gnu-sed로 gsed를 쓰는 방법도 있습니다.

LX-09

네트워크·HTTP

API를 때려보고, 포트 연결을 확인하고, DNS를 추적하는 일상 도구들. 백엔드라면 curl은 거의 반사신경이 돼야 합니다.

명령의미 / 형태
curl -i URLHTTP 요청 — -i 응답 헤더 포함. -X POST -d '{}' -H "Content-Type: application/json"
curl -s URL | jq조용히(-s) 받아 jq로 JSON 예쁘게/추출
curl -O URL파일 다운로드(원래 이름). wget URL도 같은 용도
dig example.comDNS 조회 (A 레코드·TTL). +short 간단히
ping host도달성·왕복시간 확인 (Ctrl-C로 종료)
ss -tlnp / lsof -i열린 포트·리스닝 소켓. macOS는 lsof -i -P가 흔함
nc -zv host 5432특정 포트 연결 가능한지 점검 (DB·서비스 도달성)
디버깅 순서

"서비스가 안 붙어요"의 추적 사다리: ① ping으로 호스트 도달 → ② nc -zv host port로 포트 열림 → ③ curl -v URL로 HTTP 응답·TLS 핸드셰이크 → ④ dig로 DNS가 맞는 IP를 주는지. 위에서 아래로 한 칸씩 좁히면 어디서 끊겼는지가 드러납니다(→ 백엔드 핸드북 BE-08).

LX-10

디스크·시스템

"디스크가 꽉 찼어요", "메모리가 부족해요"의 진단 도구들. 평소엔 안 쓰다가 사고 날 때 찾게 됩니다.

명령의미 / 형태
df -h디스크 여유 공간 (사람이 읽는 단위). 파티션별 사용률
du -sh *현재 위치 항목별 용량 합계 — 뭐가 용량을 먹는지 추적
du -sh * | sort -rh | head큰 것부터 정렬해 상위만 (용량 범인 색출)
free -h메모리 사용량 (Linux). macOS는 vm_stat / top
uname -aOS·커널 정보
uptime가동 시간·부하 평균(load average)
date현재 시각. 로그 타임스탬프·스크립트에서 활용
LX-11

압축·아카이브

tar의 플래그는 한 번 외우면 평생 씁니다 — "create / extract"의 두 패턴만 손에 넣으세요.

명령의미
tar -czf out.tar.gz dir/묶고 압축 — create z(gzip) file
tar -xzf out.tar.gz풀기 — extract. -C 경로로 풀 위치 지정
tar -tzf out.tar.gz풀지 않고 내용 목록만 (t)
zip -r out.zip dir/zip으로 묶기 (재귀). 윈도/공유용
unzip out.zipzip 풀기. -d 경로로 위치 지정
gzip file / gunzip file.gz단일 파일 압축/해제 (원본 대체)
기억법

tar = Tape ARchive. 외울 조합은 둘뿐 — 만들 땐 czf, 풀 땐 xzf. "compress / extract, zip(gzip), file". .tar.gz가 가장 흔하고, 더 작게는 -cJf(xz)도 있습니다.

LX-12

원격·동기화

서버에 접속하고, 파일을 옮기고, 디렉터리를 동기화. 배포·운영의 기본 동선입니다.

명령의미 / 형태
ssh user@host원격 셸 접속. -p 포트, -i key.pem(키 지정)
ssh user@host '명령'접속 없이 원격에서 명령 한 번 실행
scp file user@host:/path원격으로 파일 복사. -r 디렉터리. 반대 방향도 가능
rsync -avz src/ host:dst/변경분만 동기화 — 빠르고 재개 가능. -a 보존 -v 로그 -z 압축
rsync -avzn ...-n(dry-run) — 실제로 옮기기 전 무엇이 바뀔지 미리보기
rsync의 끝 슬래시 함정

rsync src/ dst/(끝 /)는 src 안의 내용을 dst로, rsync src dst/(끝 / 없음)는 src 디렉터리째 dst 안으로 — 결과가 다릅니다. 큰 동기화는 항상 -n(dry-run)으로 먼저 확인하는 습관이 사고를 막습니다. SSH 키는 ~/.ssh/config에 호스트 별칭을 등록해두면 ssh myserver로 짧게 씁니다.
