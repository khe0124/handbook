vi / Vim 단축키
핸드북

vi는 단축키 목록이 아니라 언어다 — 모드 위에서 "연산자 × 모션"을 조합한다. 이 문법 하나를 잡으면 외울 게 덧셈이 아니라 곱셈으로 줄어든다. d(삭제)와 w(단어)·i"(따옴표 안)를 알면 dw·di"가 공짜로 나온다. macOS 기본 vim 기준.

START : 터미널에서 vimtutor (30분, 한국어 vimtutor ko) — 이 문서는 그 다음의 빠른 참조
COMPILED : 2026-06
VI-01

사고 모델 — 모드와 문법

vi를 어렵게 만드는 건 단축키 개수가 아니라 두 가지 개념이다 — (1) 모드, (2) "동사(연산자) + 명사(모션/객체)" 문법. 이 둘을 잡으면 나머지는 조합이다.

THE GRAMMAR — 명령은 문장이다 [ 횟수 ] 연산자 [ 횟수 ] 모션/객체
예: dw = 단어 삭제  |  2dw = 두 단어 삭제  |  ci" = 따옴표 안 바꾸기
└ 연산자 d·c·y 한 줌 × 모션·객체 한 줌 = 수백 개 명령이 곱셈으로 생성된다

핵심 통찰: vi의 명령은 영어 문장처럼 읽힌다. d(delete) + w(word) = "단어를 지워라". c(change) + i"(inner quotes) = "따옴표 안을 바꿔라". 그래서 새 명령을 외우는 게 아니라 아는 동사와 명사를 새로 조합한다. y(복사)를 배우는 순간 yw·yi(·y$를 따로 안 배워도 쓸 수 있는 이유다.

CORE INSIGHT

외울 것은 결국 세 묶음뿐이다 — ① 연산자(d 삭제 · c 바꾸기 · y 복사), ② 모션(어디로/어디까지: w $ } G…), ③ 텍스트 객체(무엇을: iw i" i(…). 이 셋의 곱이 vi 편집의 전부다. 단축키를 "암기"하려 들면 끝이 없고, "문법"으로 보면 유한하다.

VI-02

모드와 전환

vi가 다른 에디터와 다른 단 하나의 이유 — 키가 모드마다 다른 뜻을 갖는다. Normal에서 d는 "삭제 명령"이고, Insert에서 d는 글자 'd'다.

모드역할진입
Normal (기본·집)이동·명령. vi의 본진 — 대부분 시간을 여기서Esc (또는 Ctrl-[)
Insert실제 글자 입력i a o 등 (아래)
Visual영역 선택 후 연산v V Ctrl-v
Command-line저장·종료·치환 등 ex 명령: (그리고 / ? 검색)
Insert 진입의미
i / a커서 앞 / 뒤에서 입력 (insert / append)
I / A줄 맨 앞(첫 글자) / 맨 끝에서 입력
o / O아래 / 위에 새 줄 열고 입력
길 잃으면 Esc

어디 있는지 모르겠으면 무조건 Esc를 두어 번 — Normal로 돌아온다. Normal이 집이다. 초보가 헤매는 90%는 "지금 무슨 모드인지" 감각이 없어서다. 입력이 끝나면 습관적으로 Esc로 빠져나오는 리듬을 들이면 vi가 갑자기 쉬워진다.

VI-03

이동 (모션)

모션은 그 자체로 "이동"이면서, 연산자 뒤에 붙으면 "어디까지 적용할지"가 된다 — w는 "다음 단어로 이동"이자 dw의 "단어만큼"이다. 그래서 모션을 익히면 편집까지 같이 빨라진다.

모션이동
h j k l좌 · 하 · 상 · 우 (방향키 대신 — 손이 홈 위치에 머문다)
w / b / e다음 단어 앞 / 이전 단어 앞 / 단어 끝. 대문자 W B E는 공백 기준(구두점 무시)
0 / ^ / $줄 맨 앞(0칸) / 첫 글자(공백 제외) / 줄 끝
gg / G파일 처음 / 파일 끝. 42G 또는 :42 = 42번째 줄로
{ / }이전 / 다음 문단(빈 줄 기준)
f{x} / t{x}줄에서 글자 x 까지(위) / x 직전까지. F T는 역방향. ; 반복 · , 역반복
%짝 괄호로 점프 ( ( ) { } [ ] )
Ctrl-d / Ctrl-u반 페이지 아래 / 위. Ctrl-f / Ctrl-b는 한 페이지
H / M / L화면 맨 위 / 가운데 / 맨 아래 줄로. zz 현재 줄을 화면 중앙에
실전

f는 한 줄 안 이동의 핵심이다 — "이 줄의 다음 쉼표로" = f,, 그 직전까지 지우기 = dt,(till comma). 마우스로 클릭하던 걸 f{글자} 한 번으로 대체하는 게 vi 속도의 절반이다.

VI-04

편집 — 연산자

연산자는 "무엇을 할지"(동사)다. 뒤에 모션(VI-03)이나 텍스트 객체(VI-05)를 붙여 범위를 정한다. 연산자를 두 번 치면(dd cc yy) "현재 줄 전체"가 된다.

연산자의미 / 조합 예
d삭제(잘라내기). dw 단어 · d$ 줄 끝까지 · dG 파일 끝까지 · dd 한 줄
c바꾸기(삭제+Insert 진입). cw 단어 · cc 줄 전체 · c$ 줄 끝까지
y복사(yank). yw · yy 한 줄 · y$ 줄 끝까지
p / P붙여넣기 — 커서 뒤 / 앞. 줄 단위 복사(yy)면 아래 줄/위 줄에
빠른 편집 (단일 키)의미
x / X커서 글자 삭제 / 앞 글자 삭제
r{x} / R한 글자만 x로 교체 / 덮어쓰기(Replace) 모드
s / S글자 지우고 입력 / 줄 비우고 입력 (= cc)
~ / J대소문자 토글 / 아래 줄을 현재 줄에 합치기
>> / <<줄 들여쓰기 / 내어쓰기
u / Ctrl-r / .실행취소 / 다시실행 / 직전 변경 반복(VI-09)
VI-05

텍스트 객체 — vi의 진짜 힘

모션이 "어디까지"라면, 텍스트 객체는 "무엇을"이다 — 커서가 그 안 어디에 있든 단어·따옴표·괄호 단위를 통째로 잡는다. 위치를 정렬할 필요가 없어서 압도적으로 빠르다.

객체의미 (i=inner 안쪽 / a=around 경계 포함)
iw / aw단어 / 단어+주변 공백
i" i' i`따옴표 안 내용 (a"는 따옴표까지)
i( i{ i[ i<괄호 안 (ib=i(, iB=i{). a(는 괄호까지
it / atHTML/XML 태그 안 / 태그 포함
ip / ap문단 / 문단+빈 줄
연산자 × 텍스트 객체 — 실전 조합 ci" 따옴표 안 내용 바꾸기   "old" → 커서 아무 데나 → "▮"
di( 괄호 안 통째로 삭제     foo( a, b ) → foo()
yip 문단 복사    dit 태그 안 비우기    caw 단어+공백 교체
왜 강력한가

ci" 하나로 끝나는 작업을 떠올려 보라 — 따옴표 안 어디에 커서가 있든, 시작/끝으로 이동할 필요 없이 내용이 통째로 비워지고 입력 모드로 들어간다. 함수 인자 바꾸기 ci(, JSX 태그 내용 바꾸기 cit, 변수명 바꾸기 ciw — 코드 편집의 대부분이 이 패턴 몇 개로 수렴한다. "안(i)이냐 경계 포함(a)이냐"만 구분하면 된다.

VI-06

검색·치환

검색은 가장 빠른 이동 수단이기도 하다(원하는 단어로 점프). 치환은 ex 명령(:)의 영역으로, 범위 지정이 핵심이다.

검색의미
/패턴 / ?패턴아래로 / 위로 검색 (Enter). n 다음 · N 이전(반대 방향)
* / #커서 아래 단어를 아래로 / 위로 검색 (변수 추적에 유용)
:noh검색 강조 끄기 (nohlsearch)
치환 (:s)의미
:s/old/new/현재 줄의 첫 old를 new로
:s/old/new/g현재 줄의 모든 old (g = global, 줄 내 전체)
:%s/old/new/g파일 전체의 모든 old (%=전체 범위)
:%s/old/new/gc전체 치환하되 하나씩 확인(c = confirm: y/n/a/q)
:10,20s/old/new/g10~20번째 줄만. 비주얼 선택 후 :s는 :'<,'> 범위로
특수문자

치환 패턴은 정규식이라 . * / \ & 등은 특별한 뜻을 갖는다 — 슬래시를 찾으려면 \/로 이스케이프하거나, 구분자를 바꿔 :%s#old/path#new/path#g처럼 #를 써도 된다(가독성↑).

VI-07

비주얼 모드

"먼저 선택하고, 그다음 무엇을 할지 정한다" — 연산자+모션의 순서를 뒤집은 모드. 범위가 눈에 보여서 직관적이고, 한 영역에 여러 작업을 시도할 때 좋다.

진입 / 동작의미
v / V / Ctrl-v문자 단위 / 줄 단위 / 블록(열) 단위 선택
선택 후 d y c삭제 / 복사 / 바꾸기. > < 들여쓰기, ~ 대소문자
gv직전 선택 영역 다시 선택
블록(Ctrl-v) + I/A여러 줄 맨 앞/뒤에 동시 입력 (Esc로 전 줄에 반영) — 주석 일괄 추가에
언제 비주얼, 언제 연산자

범위가 명확하면(단어·따옴표·줄 수) ci" 3dd처럼 연산자+모션이 빠르다 — 손이 덜 움직이고 .로 반복 가능. 범위가 애매하거나 눈으로 확인하며 늘리고 싶으면 비주얼이 편하다. 블록 비주얼(Ctrl-v)은 다른 에디터의 "멀티 커서" 자리를 메운다.

VI-08

파일·창·저장·종료

"vi에서 못 빠져나온다"는 농담의 답이 여기 있다 — 전부 :(command-line)에서 한다. Esc로 Normal에 온 뒤 :를 쳐라.

저장·종료의미
:w저장(write). :w 파일명 다른 이름으로
:q / :q!종료 / 저장 안 하고 강제 종료(버림)
:wq / :x / ZZ저장하고 종료 (셋 다 같음. ZZ는 Normal에서 바로)
:wa / :qa모든 창 저장 / 모두 종료 (:wqa 저장+모두 종료)
여러 파일·창·탭의미
:e 파일파일 열기(edit)
:ls · :bn · :bp · :b이름버퍼 목록 · 다음 · 이전 · 이름으로 이동
:sp / :vsp가로 분할 / 세로 분할 창
Ctrl-w 다음 h j k l창 사이 이동. Ctrl-w w 순환
:tabnew · gt / gT새 탭 · 다음 탭 / 이전 탭
"못 나가요"의 해답

지금 입력 중이라 명령이 안 먹으면 → Esc 먼저. 그다음: 저장하고 나가기 :wq, 버리고 나가기 :q!. 변경 안 했으면 :q. 이 세 개면 평생 탈출 가능하다.

VI-09

생산성 + 치트시트

vi가 "빠른 에디터"를 넘어 "증폭기"가 되는 지점 — 반복(.)과 매크로다. 한 번 잘 만든 변경을 공짜로 N번 재생한다.

증폭 도구의미
. (닷)직전 변경을 반복. ciwfoo<Esc> 후 다음 단어에서 . — vi 생산성의 핵심
{n}{명령}횟수 곱하기 — 3dd 세 줄 삭제 · 5j 다섯 줄 아래 · 2>> 두 줄 들여쓰기
q{a} … q / @a / @@레지스터 a에 매크로 녹화 / 재생 / 직전 매크로 반복. 10@a 10번
m{a} / ` a마크 a 찍기 / 마크 a로 점프 (`` 직전 위치로)
"+y / "+p시스템 클립보드로 복사 / 에서 붙여넣기 (클립보드 지원 vim)

치트시트 — 손에 먼저 붙일 조합:

줄 복제yyp
yy(줄 복사) → p(아래 붙이기). 한 줄 통째 복제.
단어 바꾸기ciw
커서가 단어 어디 있든 그 단어를 비우고 입력. 다음 단어에서 .로 반복.
따옴표/괄호 안 교체ci" · ci(
문자열·인자 내용 통째 교체. 위치 정렬 불필요.
줄 끝까지 지우기D (= d$)
커서부터 줄 끝까지 삭제. C는 같은 범위 바꾸기(c$).
줄 맨 끝에서 입력A
어디 있든 줄 끝으로 가 입력. I는 줄 첫 글자에서 입력.
아래에 새 줄o / O
아래(o)·위(O)에 빈 줄 열고 바로 입력.
전체 치환:%s/a/b/g
파일 전체 a→b. 확인하며 하려면 /gc.
들여쓰기 일괄Vjj>
비주얼 줄 선택(V) 후 범위 늘리고 >. 반복은 ..
방금 한 거 또. (닷)
직전 변경 반복. "수정 → n으로 다음 → ." 리듬이 핵심.
되돌리기u / Ctrl-r
실행취소 / 다시실행. 마음껏 실험하고 되돌려라.
학습 순서 제안

한 번에 다 외우지 마라. ① vimtutor 30분으로 모드·기본 이동 → ② i a o / Esc / :wq :q!로 생존 → ③ dd yy p x u로 일상 편집 → ④ ciw ci" ci( 텍스트 객체(여기서 속도가 폭발) → ⑤ /검색 :%s . 반복 → ⑥ 매크로. 4단계까지만 가도 대부분의 편집이 빨라진다. (셸 작업은 → 리눅스/셸 핸드북과 함께.)
