# Operations Handbook Accuracy Overhaul Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Fix the infrastructure/operations handbook so each section has accurate, section-specific operational evidence, commands, examples, and tests.

**Architecture:** Replace index-based playbook row reuse with section-owned operational data. The generator should render each section from explicit `questions`, `evidence`, `commands`, `incident`, and optional `examples` attached to that section, while the shared document-level playbook remains only a summary. Tests should verify semantic correctness, not just keyword presence.

**Tech Stack:** Node.js ESM generator, static HTML output, generated TypeScript document modules, Node built-in test runner.

### Task 4: Fix Korean Particle Rendering

**Files:**
- Modify: `scripts/generate-operations-handbooks.mjs`

- [ ] **Step 1: Add particle helper**

Add this near `escapeHtml`:

```js
const hasFinalConsonant = (value) => {
  const last = [...value.trim()].pop();
  if (!last) return false;
  const code = last.charCodeAt(0);
  if (code < 0xac00 || code > 0xd7a3) return false;
  return (code - 0xac00) % 28 !== 0;
};

const subjectParticle = (value) => (hasFinalConsonant(value) ? "은" : "는");
```
