# Backend Core Depth Hardening Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Turn the backend core handbook from a stronger roadmap draft into a genuinely substantial backend learning guide by fixing structural order, removing duplicate chapter codes, and deepening weak areas: foundations, Java/Spring, database design, deployment, and portfolio curriculum.

**Architecture:** `public/handbook/backend-engineering-handbook.html` remains the source of truth for the backend source document. `scripts/generate-engineering-bundles.mjs` injects the curated intro for `engineering-backend-core`, and generated outputs must be rebuilt from source rather than hand-edited. Semantic coverage is enforced in `scripts/handbook-html.test.mjs`.

**Tech Stack:** Static HTML handbook, Node.js generation scripts, Node test runner, Vite build.

---

## Review Findings This Plan Addresses

The current backend document is improved, but not yet objectively “충실하다” enough because:

- The nav reads as a roadmap, but the physical section order is still inherited from the older document.
- Chapter codes are duplicated: `BE-08` appears for cache and networking, `BE-13` appears for architecture and glossary, `BE-14` appears for API deep dive and portfolio.
- Java/Spring is too shallow for a Java/Spring-oriented backend path.
- Foundation topics from the referenced roadmap are incomplete: CORS, cookies/sessions, Linux/process/ports/logs, Git/PR workflow.
- DB coverage emphasizes indexes and transactions, but schema design, normalization/denormalization, joins, backup/restore, and migration tooling need more depth.
- Portfolio guidance says what evidence to produce, but not enough about week-by-week curriculum and project progression.
