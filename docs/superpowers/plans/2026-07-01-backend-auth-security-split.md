# Backend Auth Security Split Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Split backend authentication/security out of `개발 핸드북 > 백엔드 핵심` into a dedicated `백엔드 인증·보안` menu item, then deepen both documents so backend core becomes focused and backend security becomes interview- and practice-grade.

**Architecture:** The handbook uses source HTML files in `public/handbook`, generated document modules in `src/handbook/documents`, catalog entries in `src/handbook/catalog.mjs`, and generated bundles from `scripts/generate-engineering-bundles.mjs`. The split should create a new engineering bundle from `auth-security-handbook.html`, remove that source from `engineering-backend-core-handbook.html`, and shift later engineering menu numbers by one.

**Tech Stack:** Static HTML handbook documents, Node.js generation scripts, Vite build, Node test runner.
